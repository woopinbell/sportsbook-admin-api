# 000 · chore(project): admin-api 골격 초기화

## 개요

Phase 4의 7번째 repo. 본 구현 전에 **빌드·린트·문서 골격**을 세운다. 목표는 "이후 모든 dev 커밋이 `mvn verify` 한 줄로 컴파일·포맷·테스트되는" 토대를 만드는 것. 형제 service(settlement 등)의 검증된 pom/구조를 그대로 따라 일관성을 확보한다.

## 작업 순서

1. `.gitignore` — Java/Maven 표준 + **`CLAUDE.md` 라인 필수**(AI 세션 컨텍스트는 git에 안 올림).
2. `pom.xml` — Spring Boot 3.2.11 parent, Java 17, shared-protocol 0.2.0, web/security/oauth2-resource-server/validation/JPA/flyway/kafka/avro/observability/lombok/test 의존, Spotless+Checkstyle+avro 플러그인.
3. `config/checkstyle/checkstyle.xml` — 형제 repo 룰셋 복사.
4. `README.md` — 상단 영문 박스 + 한국어 본문(ADR-0016).
5. `application.yml` + `logback-spring.xml` — 포트 8090, actuator liveness/readiness, JSON 로그 프로파일.
6. `AdminApiApplication` + `docs/commits·reflection`의 `.gitkeep`.
7. `git init` 후 `git status`로 **CLAUDE.md가 ignored인지** 확인하고 커밋.

## 작업 내역

### 빌드 정의 (pom.xml)

ADR-0015 베이스라인. C/C++의 CMake/Makefile에 해당하는 게 Maven `pom.xml`인데, 핵심 차이는 **부모 POM 상속**으로 의존 버전을 한 번에 정렬한다는 점이다:

```xml
<parent>
  <groupId>org.springframework.boot</groupId>
  <artifactId>spring-boot-starter-parent</artifactId>
  <version>3.2.11</version>
</parent>
```

이 부모가 Spring/Jackson/JUnit 등 수십 개 라이브러리의 호환 버전을 BOM으로 고정해 준다. 그래서 대부분의 `<dependency>`에 `<version>`을 안 쓴다(버전 충돌 지옥 회피).

admin-api 특이점: `audit_log` 한 테이블 때문에 JPA+Flyway+PostgreSQL을 넣지만, 비즈니스 저장소는 일절 건드리지 않는다. 보안을 위해 `spring-boot-starter-security` + `-oauth2-resource-server`(RS256 JWT 검증기 내장)를 넣는다.

### 포트 / 관측성

실행 포트는 8090(다른 service: wallet 8081·betting 8082·risk 8083·settlement 8084·odds-feed 8085·gateway 8080과 충돌 회피). actuator는 health/prometheus만 노출하고 readiness/liveness probe를 분리(ADR-0007).

### CLAUDE.md 제외 확인

```sh
git check-ignore CLAUDE.md   # → CLAUDE.md (ignored)
```

이게 핵심 게이트. CLAUDE.md는 프로젝트 산출물이 아니라 AI 세션 운영 파일이라 어떤 커밋에도 안 들어간다.
